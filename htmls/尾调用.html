尾调用优化 Demo（可在 DevTools 观察调用栈）

说明：大多数现代 JS 引擎并没有实现规范里提到的“严格模式尾调用优化 (TCO / PTC)”。 所以「普通递归」和「尾递归写法」通常都会随着 n 增大而栈溢出； 但「trampoline / 迭代」能真正避免调用栈增长。

建议：打开 DevTools → Performance 录制 → 点下面按钮运行 也可在 Console 里看 console.profile 结果
观察点：递归两种一般会抛 Maximum call stack size exceeded； trampoline 会成功返回结果且调用栈不会随着 n 成比例增长。
