查看文章，已经在合适的地方插入了配图，

问题， 但生成图片过大， 当前这类文章配图是 PNG，尺寸 1536 x 864，又有手绘纹理和渐变色，PNG 很容易到 1MB+。

如果直接通过命令行把图片压缩，就可以完全的自动化

gpt给出了三套方案：

建议用 WebP 转码 + 质量压缩，这是最适合你这个 Docusaurus 知识库文章配图的方案。

  优先级建议：

  1. cwebp
      - Google 官方 WebP 工具
      - 压缩效果稳定
      - 适合批处理
      - 推荐作为项目规范

     示例：

     cwebp -q 82 input.png -o output.webp

  2. imagemagick
      - 更通用，能 resize、转格式、批处理
      - 如果已经装了 magick，也很好用

     示例：

     magick input.png -quality 82 output.webp

  3. sharp
      - Node 项目里很常用
      - 适合写成 npm script 或批处理脚本
      - 如果以后想自动化，可以用它

     示例：

     import sharp from 'sharp';

     await sharp('input.png')
       .webp({ quality: 82 })
       .toFile('output.webp');

我选择2， 使用 https://imagemagick.org/download/#gsc.tab=0 进行批处理压缩图片， 可以看到图片从1.3mb压缩到了56kb

## 编辑工作流

- Markdown 文件通过 **Obsidian** 编辑
- 图片由 Obsidian 自动存放至 `static/docs/` 目录
- Obsidian 粘贴图片沿用现有相对路径风格，例如 `../../static/docs/<图片文件名>`
- AI 生成的文章配图统一放在 `static/docs/aiRender/<文章分类>/` 目录下，例如 `static/docs/aiRender/应用上架与生态/`
- AI 生成配图最终使用 WebP；PNG 只作为临时源图
- 使用 ImageMagick 压缩为 WebP，默认质量 `82`，例如：`magick example.png -quality 82 example.webp`
- WebP 生成并验证成功后删除对应 PNG 源图；只有 WebP 生成失败或需要排查问题时才保留 PNG
- 默认不保存生图提示词或构思卡存档；只有用户明确要求保存时，才把提示词/构思卡写入文件
- 在文章中引用 AI 生成配图时沿用现有相对路径风格，例如 `![](../../static/docs/aiRender/应用上架与生态/example.webp)`；不要把这类配图放到 `static/img/` 后再用 `/img/...` 引用
