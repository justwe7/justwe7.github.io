问题， 但生成图片过大， 当前这类文章配图是 PNG，尺寸 1536 x 864，又有手绘纹理和渐变色，PNG 很容易到 1MB+。

如果直接通过命令行把图片压缩，就可以完全的自动化

gpt给出了三套方案：
```
建议用 WebP 转码 + 质量压缩，这是最适合你这个 Docusaurus 知识库文章配图的方案。

  优先级建议：

  1. cwebp
      - Google 官方 WebP 工具
      - 压缩效果稳定
      - 适合批处理
      - 推荐作为项目规范

     示例：

     cwebp -q 82 input.png -o output.webp

  2. imagemagick
      - 更通用，能 resize、转格式、批处理
      - 如果已经装了 magick，也很好用

     示例：

     magick input.png -quality 82 output.webp

  3. sharp
      - Node 项目里很常用
      - 适合写成 npm script 或批处理脚本
      - 如果以后想自动化，可以用它

     示例：

     import sharp from 'sharp';

     await sharp('input.png')
       .webp({ quality: 82 })
       .toFile('output.webp');
```
我选择2，
使用 https://imagemagick.org/download/#gsc.tab=0 进行批处理压缩图片， 可以看到图片从1.3mb压缩到了56kb
