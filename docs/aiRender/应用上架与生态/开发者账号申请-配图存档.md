# 开发者账号申请 - 配图存档

## 插图说明清单

1. `developer-account-payment-flow-v2.webp`
   - 插入位置：`Wise 虚拟账户` 代码块之后。
   - 画面：两个应用商店收入入口汇入虚拟账户，再回到国内银行卡，背剑侠客在中间连接收款路径。
   - 作用：解释 Apple Developer 和 Google Play 做付费 / 内购时，通常需要境外收款路径，Wise 类虚拟账户可以作为中转。

2. `developer-account-platform-compare-v2.webp`
   - 插入位置：两平台对比速览表之后。
   - 画面：苹果与谷歌两侧账号门对比，谷歌侧增加测试阶段、海外卡、IP / 设备风险，底部用分叉表示合规账号与风险账号隔离。
   - 作用：区分两平台注册与账号管理难度，并把“是否隔离账号取决于 App 风险”可视化。

## 构思卡摘要

### developer-account-payment-flow-v2

- 图型：流程图
- 配图任务：解释
- 画幅：16:9 横向，最终 WebP 为 `1536x864`
- 单一命题：付费和内购不是只注册账号，还要准备可接收海外平台付款的路径。
- 原文指纹：Apple Developer、Google Play、Wise 虚拟美国账户、W-8BEN、资金转回国内。
- 信息单位：两个商店入口、虚拟账户、税表、国内银行卡。
- 组织规则：左到右收入流向。
- 重复元素预算：只保留两个商店入口、一个虚拟账户、一个银行终点。
- 视觉重量：主流程约占画面 55%，侠客约 7%，单个模块不超过 18%。
- 手写标签：`商店`、`虚拟账户`、`税表`、`回款`。
- 阅读顺序：先看两个商店入口，再看虚拟账户中转，最后看回款到银行卡。
- 侠客动作：站在虚拟账户旁，把两条收入线合并为一条回款线。
- 一秒读图目标：看出钱从两个平台汇入一个中转账户。
- 三秒读图目标：理解境外收款路径是开发者账号准备的一部分。
- 删除项：真实平台 Logo、银行卡号、完整税务表单和复杂银行系统。

### developer-account-platform-compare-v2

- 图型：对比图 / 关系图
- 配图任务：区分
- 画幅：16:9 横向，最终 WebP 为 `1536x864`
- 单一命题：Apple 注册更顺，Google 在付款、新账号测试和多账号识别上有更多风险点，账号策略要按 App 风险决定。
- 原文指纹：$99 年费、$25 一次性、国内信用卡、海外卡、新账号测试阶段、同电脑 / IP 多账号识别、账号隔离。
- 信息单位：左右平台门、费用标签、付款卡、测试桥、IP / 设备风险、隔离分叉。
- 组织规则：左右对比加底部策略分叉。
- 重复元素预算：只保留两列平台对比和一个小型分叉，不画大量账号或设备。
- 视觉重量：左右平台视觉重量接近，策略分叉约占 20%，侠客约 7%。
- 手写标签：`苹果`、`谷歌`、`测试`、`隔离`、`合规`。
- 阅读顺序：先看左右平台门，再看谷歌侧额外风险，最后看底部账号策略分叉。
- 侠客动作：站在分叉处放置隔离挡板。
- 一秒读图目标：看出 Apple 和 Google 的门槛不一样。
- 三秒读图目标：理解多账号是否隔离取决于内容风险，而 Google 更需要谨慎。
- 删除项：真实 Logo、完整费用数字、长说明文字、浏览器截图和账号购买渠道。
