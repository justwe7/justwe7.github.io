## 构思卡摘要

### 01 - 配图规则落地

- 图型：架构图
- 画幅：16:9 横向
- 配图任务：解释
- 单一命题：Skill 负责理解和生成图片，项目规则负责决定图片如何落地到知识库。
- 原文指纹：`article-metaphor-illustrator`、`AGENTS.md`、`static/docs/aiRender/`、相对路径引用。
- 信息单位：Skill 卷轴、规则文档、aiRender 目录、Markdown 文章引用。
- 组织规则：左侧输入，中间目录，右侧文章引用。
- 重复元素预算：最多两个文档对象，规则细节用短线集合表示。
- 视觉重量：目录约 18%，输入各低于 12%，文章页低于 15%，侠客约 6%-8%。
- 手写标签：Skill、规则、目录、引用。
- 阅读顺序：先看 Skill 和规则两个输入，再看目录，最后理解文章引用的是目录中的 WebP。
- 侠客动作：把 WebP 图片卡放进目录，强调生成结果进入项目规则约束的位置。
- 一秒读图目标：看出图片从规则化目录进入文章引用。
- 三秒读图目标：看懂 Skill 和 AGENTS.md 是两个互补环节。
- 删除项：不画完整 AGENTS.md 内容，不画所有路径层级，不画多个文章分类目录。

### 02 - PNG 转 WebP 发布

- 图型：对比图加转换路径
- 画幅：16:9 横向
- 配图任务：区分
- 单一命题：PNG 适合作为生成阶段的临时源图，WebP 更适合作为网页发布图片。
- 原文指纹：PNG、WebP、ImageMagick 压缩、`quality 82` 的发布取舍。
- 信息单位：重 PNG 文件、压缩机、轻 WebP 文件、发布勾选。
- 组织规则：左到右转换路径。
- 重复元素预算：两个图片文件卡，一个压缩机，一个侠客，不复制更多文件。
- 视觉重量：PNG 文件约 16%，压缩机低于 14%，WebP 文件约 12%，侠客约 6%-8%。
- 手写标签：PNG、WebP、压缩、发布。
- 阅读顺序：先看沉重 PNG，再看压缩动作，最后看轻量 WebP 和发布勾选。
- 侠客动作：转动压缩机手柄，参与“压缩”这一关键动作。
- 一秒读图目标：看出大文件变小文件。
- 三秒读图目标：看懂正文最终引用 WebP，而不是 PNG。
- 删除项：不画具体字节数、不画工具对比表、不画 cwebp 和 sharp。

### 03 - 笔记配图六步流程

- 图型：流程图
- 画幅：16:9 横向
- 配图任务：解释
- 单一命题：给笔记配图应按文章、构思、PNG、WebP、引用、构建的顺序稳定执行。
- 原文指纹：Markdown 文章、配图 Skill 构思、PNG 源图、WebP 转换、Markdown 引用、`npm run build`。
- 信息单位：六个流程节点和一条主路径。
- 组织规则：从左到右的连续流程。
- 重复元素预算：六个小节点，内部图标不同，避免同形卡片阵列。
- 视觉重量：单节点低于 9%，整体核心约 55%，侠客约 6%-8%。
- 手写标签：文章、构思、PNG、WebP、引用、构建。
- 阅读顺序：先看橙色主路径，再看六个节点，最后理解构建验证是流程末端。
- 侠客动作：在构建验证节点勾选最终结果。
- 一秒读图目标：看出一条完整工作流。
- 三秒读图目标：看懂配图不是生成即结束，而是要落盘、引用和构建验证。
- 删除项：不画完整 shell 命令，不展开每一步的检查清单，不画多张示例图片。
